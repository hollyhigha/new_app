### 1.1 账号注册（提前办理，审核周期长）

| 平台 | 网址 | 费用 | 审核周期 |

| 小米开放平台 | dev.mi.com | 免费 | 1-3 个工作日 |

> **建议**：优先注册华为和小米，覆盖国内最大市场份额。

### 1.2 必备资质材料

**个人开发者（最低要求）：**
- 身份证正反面照片
- 手持身份证照片（部分平台要求）
- 银行卡（收款用）

**企业开发者（建议，医美类审核更顺利）：**
- 营业执照（含医疗相关经营范围，如"健康咨询"）
- 法人身份证
- 企业对公账户

### 1.3 软件著作权（强烈建议提前申请）

- **用途**：部分应用商店要求提供软著，医疗健康类尤其严格
- **申请周期**：约 1-3 个月（可加急缩短至 1-2 周，费用约 500-2000 元）
- **申请机构**：中国版权保护中心 (ccopyright.com.cn) 或委托代理机构
- **所需材料**：源代码前后各 30 页打印件、操作说明书、申请表

## 二、应用配置

### 2.1 manifest.json 关键配置

在 HBuilderX 中打开 `manifest.json` → 可视化界面配置：

```
基础配置：
  应用名称：靓人科普
  应用版本名称：1.0.0
  应用版本号：1（整数，每次发布必须递增）
  AppID：HBuilderX 自动生成，保持不变

App 图标：
  见 2.3 节

App 权限：
  见 2.4 节
```

包名建议格式：`com.公司拼音.liangren`，例如 `com.yourcompany.liangren`  
**包名一旦发布不可修改**，请谨慎确认。

### 2.2 版本号规则

```
版本名称（versionName）：给用户看，如 1.0.0 / 1.1.0
版本号（versionCode）：整数，每次提交必须比上一次大，如 1, 2, 3...
```

### 2.3 应用图标尺寸要求

需要准备以下尺寸的 PNG 图标（无透明背景，建议使用品牌粉色 #E91E63）：

**Android：**
- 48×48、72×72、96×96、144×144、192×192 px

**iOS：**
- 20×20、29×29、40×40、60×60、76×76、83.5×83.5、1024×1024 pt（@1x，Xcode 会自动生成 @2x @3x）

**应用商店展示图（截图）：**
- Android：至少 2 张，建议 4-6 张，尺寸 1080×1920 px
- iOS：5.5 英寸（1242×2208）和 6.5 英寸（1242×2688）各至少 1 张

### 2.4 权限声明（最小化原则）

在 `manifest.json` → App 权限配置中，**只勾选以下权限**：

```
必选：
  android.permission.INTERNET                    # 网络访问
  android.permission.ACCESS_NETWORK_STATE        # 检测网络状态

按需选：
  android.permission.READ_PHONE_STATE            # 设备标识（统计用，可选）
  android.permission.READ_CLIPBOARD              # 剪切板读取（click_id 获取用）
```

> **注意**：不要勾选摄像头、麦克风、位置等敏感权限，会影响审核通过率。

### 2.5 启动页配置

- 尺寸：1080×1920 px（竖屏）
- 背景色：白色或品牌色 #E91E63
- 展示时长：建议 2-3 秒
- 内容：App Logo + Slogan（如"专业医美科普 · 靓人科普"）

## 三、Android 打包与上架

### 3.1 生成签名证书

在本机执行（需安装 JDK）：

```bash
keytool -genkey -alias liangren -keyalg RSA -keysize 2048 \
  -validity 36500 -keystore liangren.keystore
```

按提示填写信息，设置密码（**务必保存好密码和 .keystore 文件，丢失无法找回**）。

### 3.2 HBuilderX 云打包

1. 菜单 → 发行 → 原生 App-云打包
2. 选择 Android 平台
3. 填写签名信息：
   - 证书别名：liangren（上一步设置的 alias）
   - 证书私钥密码：上一步设置的密码
   - 证书文件：选择 liangren.keystore
4. 点击"打包"，等待云端编译（约 5-15 分钟）
5. 下载生成的 .apk 文件

### 3.3 各平台提交

**华为应用市场（AppGallery）**

1. 登录 developer.huawei.com → 我的应用 → 新建应用
2. 填写应用信息：名称、分类（医疗健康）、简介
3. 上传 APK
4. 上传截图（至少 3 张）
5. 填写隐私政策 URL（必须填写，见第五节）
6. 提交审核

**小米应用商店**

1. 登录 dev.mi.com → 应用服务 → 管理应用
2. 新建应用 → 上传 APK → 填写信息
3. 需提供《软件功能说明》文档（描述 App 主要功能）
4. 医疗健康类可能需要额外提供资质证明

**应用宝（腾讯）**

1. 登录 open.qq.com
2. 新增应用 → 按流程上传
3. 需要通过微信开放平台关联（如使用微信登录则必须关联）

### 3.4 Android 审核常见被拒原因

| 被拒原因 | 解决方案 |
|---------|---------|
| 无隐私政策 | 确保隐私政策 URL 可正常访问 |
| 权限与功能不符 | 删除未使用的权限声明 |
| 医疗内容未声明 | 添加免责声明，说明内容仅供参考 |
| 无软著/资质 | 提前申请软著，或提供合作机构资质 |
| 崩溃/闪退 | 在多型号真机上充分测试 |

## 四、iOS 打包与上架

### 4.1 申请苹果开发者账号

1. 访问 developer.apple.com → 加入 Apple Developer Program
2. 费用：个人/企业均为 $99/年（约 700 元）
3. 企业账号审核需要 D-U-N-S 编号（邓白氏码），申请免费但需 1-2 周
4. 个人账号无法发布企业内部分发应用，发布到 App Store 可选个人或企业

### 4.2 证书与描述文件

在 Xcode 或 developer.apple.com 中创建：

1. **App ID**：填写包名，如 `com.yourcompany.liangren`
2. **证书（Certificate）**：
   - 开发证书（Development）：用于真机调试
   - 发布证书（Distribution）：用于 App Store 提交
3. **描述文件（Provisioning Profile）**：
   - App Store Distribution Profile：绑定 App ID + 发布证书

将证书（.p12）和描述文件（.mobileprovision）下载后在 HBuilderX 云打包时配置。

### 4.3 HBuilderX iOS 云打包

1. 菜单 → 发行 → 原生 App-云打包
2. 选择 iOS 平台
3. 上传 .p12 证书 + 输入证书密码
4. 上传 .mobileprovision 描述文件
5. 打包完成后得到 .ipa 文件

### 4.4 提交 App Store

1. 使用 Xcode 或 Transporter（苹果官方工具）上传 .ipa
2. 登录 appstoreconnect.apple.com
3. 创建新 App → 填写信息：
   - 名称：靓人科普
   - 副标题：专业医美科普知识平台
   - 关键词：医美、整形科普、皮肤管理、双眼皮（最多 100 字符）
   - 描述：500-4000 字，详细描述 App 功能
   - 分类：医疗 / 健康健美
4. 上传截图（必须提供 6.5 英寸机型）
5. 填写隐私政策 URL
6. 隐私数据说明（新要求）：声明收集的数据类型
7. 提交审核（通常 1-3 个工作日）

### 4.5 App Store 审核常见被拒原因

| 被拒条款 | 解决方案 |
|---------|---------|
| 4.0 设计糟糕 | 确保 UI 完整，无空白页面 |
| 5.1.1 隐私数据 | 填写完整的数据收集说明 |
| 1.2 用户信息 | 隐私政策必须详细且可访问 |
| 医疗健康内容 | 添加明确免责声明，说明非诊疗 |
| 账号功能不完整 | 确保注册/登录/注销流程完整 |

## 五、合规要求

### 5.1 隐私政策部署

App 内已有本地隐私政策内容，但各大应用商店**要求提供可访问的 URL**，不能是本地页面。

**部署步骤：**

1. 将 `pages/webview/index.vue` 中的 `privacyHtml` 内容保存为 HTML 文件
2. 部署到任意可公网访问的服务器，例如：
   - uniCloud 静态托管（免费）：在 uniCloud 控制台 → 静态网站托管 → 上传 HTML 文件
   - 或托管到 GitHub Pages（免费）
3. 获得 URL 后填入各应用商店

**推荐格式：**
```
隐私政策：https://你的域名/privacy.html
用户协议：https://你的域名/agreement.html
```

### 5.2 个人信息收集清单（各应用商店要求）

根据现有代码，需声明收集以下信息：

| 信息类型 | 收集场景 | 用途 | 是否可拒绝 |
|---------|---------|------|---------|
| 手机号码 | 注册/登录 | 账号验证 | 是（不登录也可使用） |
| 姓名 | 填写咨询表单 | 匹配咨询服务 | 是 |
| 城市 | 填写咨询表单 | 匹配本地机构 | 是 |
| 设备标识符 | App 启动 | 统计分析 | 否（系统权限） |
| 剪切板 | App 启动（一次性） | 获取广告来源参数 | 是 |

### 5.3 第三方 SDK 声明

需在隐私政策中声明使用的第三方 SDK：

| SDK 名称 | 提供方 | 用途 | 收集信息 |
|---------|------|------|---------|
| uni-app 运行时 | DCloud | App 框架 | 设备信息 |
| uniCloud | DCloud/阿里云 | 云服务 | 请求日志 |
| 阿里云短信 | 阿里云 | 验证码发送 | 手机号 |

### 5.4 医疗健康类免责声明

App 内已有免责声明，确认以下内容存在于显著位置：

- 首页或关于页面：**"本平台内容仅供科普参考，不构成医疗建议，具体诊疗请咨询专业医生"**
- 案例展示页：**"以下案例仅供参考，效果因人而异，请以面诊结果为准"** ✅（已有）
- 表单提交前：**"提交即授权将信息提供给合作医疗机构"** ✅（已有）

## 六、云服务配置

### 6.1 短信验证码（登录功能）

1. 登录阿里云控制台 → 短信服务 → 申请签名和模板
   - 签名：靓人科普（需提供相关资质）
   - 模板：`您的验证码为${code}，5分钟内有效，请勿泄露`
2. 在 uniCloud 控制台 → 服务空间 → 扩展服务 → 开启短信服务
3. 在 `uni-config-center` 中配置：

```json
// uniCloud-aliyun/cloudfunctions/common/uni-config-center/uni-id/config.json
{
  "sms": {
    "name": "靓人科普",
    "codeExpiresIn": 300,
    "aliyunSms": {
      "accessKeyId": "你的 AccessKeyId",
      "accessKeySecret": "你的 AccessKeySecret",
      "signName": "靓人科普",
      "templateCode": "SMS_xxxxxxxx"
    }
  }
}
```

### 6.2 企业微信 webhook 通知

1. 在企业微信群中：群设置 → 群机器人 → 添加机器人 → 复制 Webhook URL
2. 在 uniCloud 控制台 → 云函数 → `notify-wechat` → 环境变量：

```
INTERNAL_WEBHOOK_URL = https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxxxxxxx
```

### 6.3 数据库初始化

确认以下集合已在 uniCloud 控制台创建：
- `leads`：线索表（参考 `leads.schema.json`）
- `clients`：客户表
- `conversion-logs`：回传日志表
- `uni-id-users`：用户表（uni-id 自动创建）

## 七、巨量引擎回传配置

### 7.1 整体流程

```
用户点击抖音广告
    ↓
进入应用商店下载页（带 click_id 参数）
    ↓
用户安装 App 并首次启动
    ↓
App 从 URL Scheme / 剪切板中读取 click_id
    ↓
用户同意隐私协议后 → 回传"激活"事件
    ↓
用户填写咨询表单提交 → 回传"表单提交"事件
```

### 7.2 获取 Access Token

1. 登录巨量引擎后台（ad.oceanengine.com）
2. 工具 → 转化追踪 → 查看 Access Token
3. 或通过 API 授权：工具 → 授权管理 → 获取 APP Access Token

### 7.3 配置环境变量

在 uniCloud 控制台 → 云函数 → `report-conversion` → 环境变量：

```
OE_ACCESS_TOKEN = 你的巨量引擎 Access Token
```

同样为 `retry-conversion` 云函数添加相同的环境变量。

### 7.4 配置定时重试

失败的回传会在数据库中记录，需要配置定时任务触发重试：

在 uniCloud 控制台 → 云函数 → `retry-conversion` → 定时触发：
```
cron 表达式：0 */10 * * * *   （每10分钟执行一次）
```

### 7.5 回传事件说明

| 事件名 | 巨量引擎事件类型 | 触发时机 | 代码位置 |
|--------|--------------|---------|---------|
| app_install | 激活（type=4） | 用户同意隐私协议后 | `App.vue` → `handleClickId` |
| form_submit | 表单提交（type=6） | 用户提交咨询表单后 | `submit-lead` 云函数 |
| register | 注册（type=2） | 用户首次注册成功 | `utils/login-hook.js` |

### 7.6 回传验证

配置完成后，在巨量引擎后台验证：
1. 转化追踪 → 数据报表 → 查看回传数量
2. 预期：每次表单提交后，`conversion-logs` 集合中应新增一条 `success: true` 的记录

## 八、上架后运营

### 8.1 线索查看

线索数据存储在 uniCloud `leads` 集合中，可通过以下方式查看：
- uniCloud 控制台 → 数据库 → leads 集合 → 数据浏览
- 或开发简单的管理后台（后续迭代）
- 企业微信机器人会即时推送新线索通知

### 8.2 版本迭代流程

```
修改代码 → HBuilderX 云打包（版本号 +1）
→ 各应用商店提交新版本 → 等待审核
→ 审核通过后用户自动收到更新提示
```

App 内置了自动更新检测（`check-update` 云函数），可在 uniCloud 数据库中配置强制更新。

### 8.3 数据监控

建议上线后关注：
- uniCloud 控制台 → 云函数日志（查看有无报错）
- `conversion-logs` 集合中 `success: false` 的记录（回传失败）
- `leads` 集合的新增趋势
