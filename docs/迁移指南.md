## 零、未上架项目的极简迁移（当前场景）

> **当前项目状态**：代码已开发完成、uniCloud 资源在**个人账号**下、**尚未在任何应用商店上架**、**公司主体待确认**。
>
> 这是最理想的迁移场景 —— **没有用户、没有评论、没有备案、没有证书**需要继承，所以整个迁移退化为一次"代码 + 云函数 + 种子数据的搬家"，**1-2 天**即可完成。

### 0.1 "现在该做 / 不该做"一张表（最重要！）

公司主体确认之前，**以下事项建议暂缓**，否则要么做了白做（还得转让），要么白花钱。

#### ❌ 现在不要做

| 项 | 原因 | 预估白花成本 |
|---|---|---|
| 以个人主体申请 **Apple Developer** | $99 年费 + 个人账号上不了医美类 App | $99 |
| 以个人主体注册 **华为 / 小米 / OPPO / vivo / 应用宝 / 荣耀** 开发者 | 医美类要求企业主体，且账号不能转让 | 时间成本 |
| 以个人主体办理 **软件著作权** | 后续转让公司要 150~1000 元 + 30 个工作日 | 200 + 一轮等待 |
| 以个人主体办理 **ICP 备案 / App 备案 / 公安备案** | 主体变更需注销重备，个人主体备案医美相关域名容易被驳 | 15~30 工作日 |
| 以个人主体申请 **阿里云短信签名** | 医美类短信签名要企业资质 | 1-3 工作日 |
| 开通 **巨量引擎广告主** 并投放 | 跨主体不能转，转广告主数据会丢 | 真金广告费 |
| 启用 **微信开放平台**（应用宝关联） | 主体变更要 300 元/次 | 300 元 |
| 正式购买 **"靓人科普" 相关商标** | 可以以自己名义先注册，但注册成功后得走"商标转让"（≈500 元 + 6 个月） | 500 元 |
| 上线前在 **HBuilderX 云打包生成的 IPA** 提交 TestFlight / App Store | 同上，账号不对 | — |

#### ✅ 现在可以 / 应该做（不会白做）

| 项 | 说明 |
|---|---|
| 用个人 DCloud 账号**继续开发、本地真机调试** | AppID 可以无痛转让，代码资产不会损失 |
| 个人 uniCloud 空间**做功能测试** | 新公司空间建好后，种子数据会重新灌入，老空间随时弃用 |
| 打包"自测包"（Android Debug 签名 / iOS Ad Hoc）做内部测试 | 这些签名不是正式上架证书，扔了也不心疼 |
| 准备**商标检索报告**（花 0~200 元先查） | 确认 "靓人科普" 四个字是否可注册，避免公司主体确认后才发现冲突 |
| **确定品牌资产**：App 名、Logo 源文件（AI/PS）、配色、Slogan | 确认后由公司主体一次性注册商标、登记软著 |
| 准备**隐私政策 / 用户协议的最终文案**（模板已在 `pages/webview/index.vue`） | 公司主体确认后直接改主体信息即可 |
| 调研合作医疗机构（收集资质复印件） | 到时候是公司和机构签协议 |
| 调研 **商标代理、软著代理、ICP 备案服务商** 的报价 | 公司主体一确认立即走，节省工期 |

|---|---|---|

| T+3 | **新公司 DCloud 账号**注册（公司邮箱 + 手机） | 研发 |
| T+3 | **新 uniCloud 服务空间**创建 | 研发 |

| T+5 | 启动 **ICP 备案**（阿里云代办，约 15 工作日） | 运营 |
| T+5 | 开始 **阿里云短信签名/模板**审核（ICP 备案号下来后，1-3 工作日） | 运营 |
| T+5 | 启动 **公安备案**（30 工作日，不影响上架，后置做） | 运营 |

### 0.3 技术侧迁移（T+5 日，1 人 1 天搞定）

> 把"个人 uniCloud 空间 + 代码"搬到"新公司 uniCloud 空间"的最短路径。由于**没有业务数据**，可以跳过数据导出/导入。

**步骤：**

```bash
# 1. 把代码推送到公司 Git 仓库（如 GitHub Org / Gitee 企业版 / 自建 GitLab）
#    个人 Git 仓库可保留为归档
git remote add company git@github.com:<org>/liangren-kepu.git
git push company main

# 2. 本地执行种子数据生成（确保最新）
node tools/clean-mock-data.js
node tools/gen-seed-data.js
```

在 HBuilderX 里：

```
1) 新公司 DCloud 账号登录 → 右键 manifest.json → 重新获取 AppID
   ├─ 选项 A（推荐）：让新 appid（形如 __UNI__XXXXXXX）直接覆盖
   │              manifest.json 中的 appid，老个人 AppID 作废
   └─ 选项 B：个人账号转让项目给公司账号，AppID 不变

2) 右键 uniCloud-aliyun/ → 关联服务空间 → 选择新公司 uniCloud 空间

3) 上传所有 schema：
   右键 uniCloud-aliyun/database/ → 上传所有 Schema

4) 上传公共模块（先改生产配置）：
   修改 uniCloud-aliyun/cloudfunctions/common/uni-config-center/uni-id/config.json
     ├─ tokenSecret: 生成新随机串（64 字符）
     ├─ service.sms.smsKey / smsSecret: 新公司阿里云 RAM
     └─ app.smsCode.templateId: 新短信模板 ID
   右键 common/ → 上传公共模块

5) 上传所有云函数：
   右键 cloudfunctions/ → 上传所有云函数

6) uniCloud web 控制台配置环境变量：
   ├─ report-conversion / retry-conversion: OE_ACCESS_TOKEN=<新广告主 token，若还没申请则留空>
   ├─ notify-wechat / retry-conversion: INTERNAL_WEBHOOK_URL=<新企微群 webhook>
   └─ import-seeds: SEED_IMPORT_SECRET=<新随机串>

7) 配置定时触发（web 控制台 → 云函数 → 定时触发）：
   ├─ clean-expired: 每日 0:00
   └─ retry-conversion: 每 5 分钟

8) 灌入种子内容：
   控制台 → 云函数 import-seeds → 运行
   传参: { "secret": "<步骤 6 中的 SEED_IMPORT_SECRET>" }
   返回 articles.created=34 / qa.created=22 / tips.created=8 即成功

9) 代码里其他硬编码更新：
   ├─ pages/webview/index.vue → ALLOWED_HOSTS: ['liangren.com']
   │                           （改成公司新域名 + 备案域名）
   ├─ pages/webview/index.vue → privacyHtml / agreementHtml
   │                           （把"运营方""联系方式"改成公司信息）
   ├─ pages/about/index.vue（如有）→ ICP 备案号、公司名
   └─ App.vue → APP_STORE_URL: 留空，上架后回填

10) 真机回归：
    ├─ 登录（短信验证码）
    ├─ 提交留言 → 检查 uniCloud 控制台 leads 集合
    ├─ 首页/详情/搜索/发现页 都能正常加载
    ├─ 反馈 → 检查 feedbacks 集合
    └─ 设置 → 账号注销 流程可用

11) 上线前收尾：
    ├─ 删除个人 uniCloud 空间里的测试数据（但先保留空间，1 个月后再彻底删除）
    ├─ 删除个人 DCloud 账号下的 AppID（如果已经切到新 AppID）
    └─ 把个人 Git 仓库改为 private 或归档
```

### 0.4 不要忘记的"非技术迁移"动作

上面步骤只覆盖代码/云端。**以下也是"迁移"的一部分**：

- [ ] 上架指南、迁移指南、所有 docs/ 里提到的"公司名 / 联系方式"都替换为新公司
- [ ] 合作医疗机构的**协议主体**走新公司签字（如之前个人跟机构有口头/草签协议，须重签）
- [ ] 所有"靓人科普"品牌相关账号（微博、小红书、公众号、抖音企业号）**直接以新公司主体注册**，不要用个人身份先占位
- [ ] **对外宣传、BD 接触**都以新公司名义，避免埋下"应用主体与宣传主体不一致"的审核风险
- [ ] 保险柜 / 1Password 建立"靓人科普 - 公司运营"保险库，按 [上架指南 §11.4](./上架指南.md#114-证书与密钥必须入库托管) 逐项入库

### 0.5 最快能上架的时间估计

从"公司主体确认日"（T 日）算起：

- **乐观**（所有资质并行推进，不卡壳）：**T+20 工作日** 可以提交 iOS 审核
- **正常**：**T+30 工作日** 提交 iOS，再 **+3~7 天** iOS 通过，同期开始报华为/小米等安卓商店
- **保守**（含商标、公安备案也等齐）：**T+60 工作日** 全商店覆盖

医美类**最长等待点** = ICP 备案（15 工作日）+ Apple D-U-N-S（2 周）+ 短信签名审核（需等 ICP 完成）。

### 1.2 建议的迁移策略（三选一）

#### 🅰 **平滑迁移（推荐）**
- 公司账号下**新建并列资产**（新 uniCloud 空间 / 新商店应用 / 新巨量广告主）
- 现网版本继续运行 → **灰度 30 天** → 新版本全量 → 老版本进入只读模式 → 最终下架
- 用户数据通过**云函数迁移**后由用户**重新登录**触发拉取，新老数据库只读同步一段时间
- 优点：零停机、可回滚；缺点：周期长、运维复杂

#### 🅱 **窗口期迁移（≤1 天停服）**
- 选择日活低谷时段（如凌晨 2-6 点）
- 停写入 → 数据全量导出 → 切流到新空间 → 开写
- 期间用户看到"维护公告"
- 优点：数据一致性强；缺点：有停服

#### 🅲 **重启版本（应用商店强制）**
- 当商店账号不能转让时（华为/小米/OPPO/vivo/应用宝），本质是**同一个应用用新包名 / 新主体重新上架**
- 旧版本推送"请下载新版"公告，引导用户跳转新商店地址下载
- 评论、下载量清零

> 本项目推荐组合：**🅰 + 🅲** — 后端（uniCloud/巨量/短信）走平滑迁移，应用商店走重启版本。

### 1.3 决策树

```
你要迁移的是什么？
├─ 只是 uniCloud 服务空间 → 走第 4.2 节，1-2 天完成
├─ 只是某个商店账号 → 走第 4.3~4.5 节对应子章节
├─ 整体公司化（推荐模式） → 按本文档全量执行，30 天
└─ 代码/仓库转移 → 第 4.1 节 + 第 5 节
```

### 2.2 准备"主体变更授权书"模板一份（个人→公司）

以下场景会用到：

- 软著转让（需公证）
- 微信开放平台主体变更
- ICP 备案主体变更
- App 备案主体变更
- 商标权属变更（如果商标也在个人名下）

模板要点：

```
授权书

甲方（授权方/原持有人）：[个人姓名]，身份证号：[xxx]
乙方（受让方/新主体）：[公司全称]，统一社会信用代码：[xxx]

甲方同意将其持有的 [资产名称/注册号/域名/软著号] 完整、无偿转让给乙方，
```

## 四、各账号逐项迁移流程

### 4.1 DCloud（uni-app / HBuilderX）账号

> DCloud 支持**项目 AppID 转让**，无需重建。

**步骤：**

1. 新公司邮箱注册 DCloud 账号（dev.dcloud.net.cn），完成手机+邮箱双验证
2. 到 dev.dcloud.net.cn → 我的应用 → 选中"靓人科普"→ 应用详情 → **"转让"**
3. 输入接收账号（公司邮箱）
4. 对方账号接收 + 同意
5. `manifest.json` → `appid`（`__UNI__C37C42A`）**保持不变**，只是归属变更
6. HBuilderX 重新登录公司账号即可继续云打包

> ⚠️ 云打包证书：个人账号生成的 Android keystore / iOS p12 **都可以继续使用**（证书与账号非绑定），但是建议借此机会把所有密码轮换一次并存入公司 1Password。

### 4.2 uniCloud 服务空间（**最复杂**）

这是整个迁移中**工作量最大的一块**，涉及云函数、数据库、环境变量、权限策略、定时任务、公共模块。

#### 4.2.1 方案选择

- **方案 A**：服务空间直接**转移账号归属**（通过阿里云工单，仅限同类型账号；不推荐，容易卡验证）
- **方案 B（推荐）**：新公司账号下**新建空间**，代码 + 数据全部搬过去

以下以方案 B 为准。

#### 4.2.2 前置准备

- 新公司账号已开通 uniCloud（dev.dcloud.net.cn → 登录 → 新建服务空间 → 阿里云）
- 新空间 spaceId 记录下来，例如 `mp-xxxxxxxxxxxxxxx`（HBuilderX 绑定时用到）

#### 4.2.4 切流方式（客户端如何从旧空间切到新空间）

uni-app 的 `uniCloud-aliyun/` 目录绑定关系写在本地代码里，**每次云打包会把当前绑定的 spaceId 编译进去**。因此切流方式为：

1. HBuilderX 切到新空间
2. 打出新版 APK / IPA（versionCode +1）
3. 各商店发布新版
4. 老版用户仍然请求旧空间 → 旧空间**保留 30 天**只读运行
5. 30 天后关停旧空间

> 如果想做到"老版也能用新空间"，需要通过 `check-update` 强制升级拦住老版客户端，否则无法做到。

### 4.3 Apple Developer 账号

> **苹果不支持"个人账号 → 公司账号"的转让**。必须：
> 1. 新公司账号申请 D-U-N-S（1-2 周）
> 2. 注册公司 Apple Developer（$99/年，审核 2-7 天）
> 3. 公司账号下**重新创建 Bundle ID**（与个人下同名即可）
> 4. 公司账号下**重新上架应用**

#### 4.3.1 评论与下载量保留方案

苹果支持"**App Transfer**"（App 转让）功能，但要求：

- 源账号与目标账号**都是 Apple Developer 付费账号**
- 应用**无任何 In-App Purchases** 使用 wallet/apple pay 等
- 应用**没有版本处于审核中**

如果满足，则可：
1. 源账号（个人）App Store Connect → My Apps → App Transfer
2. 输入目标公司账号 Apple ID
3. 目标账号接收 → 评论、下载量、订阅关系**完整保留**

如不满足（常见于医美类 App 因账号性质被限制）：
- 只能**新账号重新上架**，评论清零，应用更名或加"新版"后缀在审核期区分

#### 4.3.2 迁移步骤（重新上架路径）

```
1. 公司注册 Apple Developer（参见上架指南 §2.3）
2. 新建 App ID：com.<公司域名反写>.liangren（同原包名）
3. 新建 Distribution 证书 & Provisioning Profile
4. HBuilderX manifest.json → iOS → Bundle ID 如改了则同步
5. 云打包 iOS（使用新公司证书）
6. Transporter 上传到新账号 App Store Connect
7. 填写 Privacy / 演示账号 / 截图等（参见上架指南 §6）
8. 提交审核
9. 新版通过后，旧版在老账号下下架（可保留历史评论可见一段时间）
```

#### 4.3.3 用户迁移提示

- 老账号版本如果还能登录：推送一条**应用内公告**"我们正在升级应用，请前往 App Store 搜索'靓人科普'下载最新版"
- 不建议直接强下架，先并行 30 天

```
   {
     "service": {
       "sms": {
```

### 4.7 巨量引擎 / 其他广告主账号

> ❌ 不支持跨主体转让，必须在新公司名下**新建广告主**。

#### 4.7.1 迁移策略

```
旧个人广告主（继续跑到停止投放为止）
     ↓
     渐进暂停所有投放计划
     ↓
新公司广告主（新建）
     ↓
新的 click_id 体系 + 新 OE_ACCESS_TOKEN
     ↓
客户端新版本携带新 Token 上传
```

#### 4.7.2 步骤

1. 登录 ad.oceanengine.com，用新公司邮箱注册广告主
2. 提交企业资质：营业执照 + 法人身份证 + 对公账户（打款验证 0.01 元）

4. 新广告主 → 工具 → 转化追踪 → 创建转化事件
   - `app_install`（event_type=4）
   - `form_submit`（event_type=6）
   - `register`（event_type=2）
5. 获取新 Access Token（长期）
6. 更新 uniCloud 云函数环境变量 `OE_ACCESS_TOKEN`
7. 在个人广告主下：把所有投放计划**暂停**（不要直接删，保留数据报表用于 30 天内对账）

#### 4.7.3 历史数据

- 个人广告主的 click_id 历史在**老 uniCloud 空间**的 `conversion-logs` 表
- 迁移期把这张表导出后**归档到 OSS**，后续用于结算/对账
- 新广告主从 0 开始累计，数据互不影响

### 4.11 工信部 App 备案 / ICP 备案 / 公安备案

> ❌ 主体变更需**注销重备**（不能直接主体变更），平均周期 15 工作日。

#### 4.11.1 顺序很重要

```
第 1 步：新公司完成 ICP 备案（域名）        ≈ 15 工作日
第 2 步：新公司完成 App 备案（工信部）       ≈ 1-2 工作日（ICP 备案完成后）
第 3 步：新公司完成公安备案                  ≈ 30 工作日
第 4 步：老主体再注销原备案                  ≈ 3-5 工作日
```

**不能反过来做**：如果先注销老备案，域名就立即进入"备案空窗期"，期间隐私政策页、官网都不可访问，将导致应用商店"隐私政策链接无效"被拒。

#### 4.11.2 ICP 备案变更要点

- 走的是对应云服务商（如阿里云）的备案系统
- 需要核验幕布 + 法人短视频（阿里云现在支持 App 拍）
- 域名持有人与备案主体必须一致（所以域名过户要在 ICP 备案前）

#### 4.11.3 App 备案要点

- 访问 **工信部 App 备案管理系统**（https://beian.miit.gov.cn）
- 备案号格式：`xx ICP备xxxxxxxx号-xxA`（App 后缀 A）
- 填写：
  - App 名称：靓人科普
  - AppID：新公司 DCloud AppID（`__UNI__C37C42A`）
  - 包名：com.liangren.kepu
  - 签名 MD5：keytool 获取
  - 开发者：新公司
  - 运营者：新公司
  - 分发渠道：苹果 + 华为 + 小米 + OPPO + vivo + 应用宝 + ...
- 每次新增渠道时需要"备案变更"

#### 4.11.4 公安备案

- 访问 **全国互联网安全管理服务平台**（www.beian.gov.cn）
- 主体变更 = 原主体注销 + 新主体新建
- 审核期较长（30 天），影响不大因为平台没有直接校验

#### 4.11.5 备案信息在 App 内的呈现

本项目**"我的 → 关于我们"页面** 需要显示以下信息，迁移后需要更新：

```
ICP 备案：<新公司 备案号>
App 备案：<新公司 App 备案号>
公安备案：<新公司 公网备案号>（可选）
运营主体：<新公司全称>
```

代码位置：`pages/about/index.vue`（如无该字段则新增）

## 七、回滚方案

### 7.1 回滚触发条件

在 T+7 全量之前，发生以下任一情况则回滚到旧环境：

- 新 uniCloud 空间连续 3 次云函数超时 / 5xx
- 新阿里云短信发送失败率 > 5%（通常是签名/模板未生效）
- 新巨量广告主 Access Token 失效 / 未授权
- 应用商店下架通知

### 7.2 回滚步骤（<30 分钟）

```
1. HBuilderX 切回旧 uniCloud 空间
2. manifest.json 退回到旧 appid（git revert）
3. 云打包 → 应用商店热更新（如 iOS 走 Meta Hotfix，Android 走自有更新通道）
4. 企业微信群通知：回滚完成
5. 分析问题并计划第二次迁移
```

### 7.3 回滚期间的数据处理

T-7 到 T+7 期间在新空间产生的**新数据**（leads / feedbacks），回滚后需要：

- 把新空间这 14 天的增量数据**反向导入**老空间
- 反向导入命令同 §4.2.3 第 9 步，但方向相反
- 通知运营：如有线索客户已经被联系，后续跟进要在老系统继续

## 九、常见坑 & FAQ

### Q1：HBuilderX 切了新 uniCloud 空间后，上传云函数报 "spaceId not found"？

A：多半是工作区没刷新。**关闭 HBuilderX 重开**，并在项目根目录右键 `uniCloud-aliyun` → 关联服务空间 → 重新选择新空间。

### Q2：阿里云短信签名审核不通过（驳回理由"资质不全"）？

A：医美类审核比较严，除了营业执照 + ICP，建议再补：
- 软著证书（新公司名下）
- 运营说明（写清"仅发送验证码，不用于营销"）
- 个人隐私政策链接（注意是新公司域名）

### Q3：工信部 App 备案填"分发渠道"时，我还没在新商店上架怎么办？

A：允许**先备案**后上架（按计划填写），但上架后要在 30 个工作日内补充备案登记号到商店后台，否则商店可能在后续巡检中下架你。

### Q4：Apple 审核员问"为什么同一个 App 出现在两个账号下"？

A：在 App Store Review Notes 里主动说明：
> The app was originally published under my personal developer account (Apple ID: xxx@xx.com). Due to business restructuring, it is now being re-submitted under our company account. The personal account version will be removed from sale within 7 days after this version is approved.

### Q5：微信开放平台主体变更期间应用宝审核卡住？

A：主体变更流程中微信开放平台会暂时锁定部分 API 调用。**建议在应用宝上架审核通过后再启动**微信开放平台主体变更。

### Q6：uniCloud 数据库导出字段类型变了（timestamp → string）？

A：导出/导入 JSON 时，`timestamp` 会被序列化为 string（Unix ms）。导入时 uniCloud schema 会按 `bsonType: timestamp` 反序列化，**一般不会出问题**。但如发生异常，可以：

```javascript
// 写个一次性云函数 fix-timestamp 做转换
const db = uniCloud.database()
exports.main = async () => {
  const { data } = await db.collection('leads').where({}).get()
  for (const item of data) {
    await db.collection('leads').doc(item._id).update({
      create_time: typeof item.create_time === 'string' ? parseInt(item.create_time) : item.create_time
    })
  }
}
```

### Q7：从个人号转让 DCloud 项目后，HBuilderX 登录一直提示 "AppID 不属于当前账号"？

A：转让后缓存未同步，请：
1. HBuilderX → 文件 → 首选项 → 清除账号缓存
2. 重新登录公司账号
3. 打开项目时右下角会提示"同步 AppID"，点击即可

### Q8：原 iOS 评论怎么保留？

A：只有 Apple 官方 "App Transfer" 可以保留评论。如果该路径不可用（常见于医美类），评论**不可迁移**。迁移后的版本从 0 评论开始，这是正常现象，不是 bug。

### Q9：巨量引擎的"广告资产（创意、素材）"能从个人广告主迁到公司广告主吗？

A：不能自动迁移，但**人工可以**：
1. 在个人广告主下导出创意素材（图片、视频）
2. 在公司广告主下重新上传素材库
3. 历史投放数据和 ROI 报表保留在个人广告主下（建议下载 CSV 存档）

### Q10：公司主体变更后，合作医疗机构授权书还有效吗？

A：**法律上不自动继承**。所有合作医疗机构的授权协议需要：
- 方案一：签订《合同主体变更协议》（原三方变新三方）
- 方案二：重新签订授权协议（新公司与每家机构）

以免被商店审核时发现"授权书上的运营方与应用主体不符"。

## 附录 A：uniCloud 数据库导出/导入命令速查

### A.1 Web 控制台方式（推荐，< 1 万条）

```
uniCloud web → 云数据库 → 选择集合 → 导出
                                       │
                                       ├─ 格式: JSON Line / JSON
                                       └─ 条件: 可选时间范围 / where
```

### A.2 命令行方式（大数据量）

uniCloud 目前没有开放 CLI，但可以写一次性云函数：

```javascript
// cloudfunctions/export-one-off/index.js
'use strict'
const db = uniCloud.database()
exports.main = async (event) => {
  const { collection, batchSize = 1000, skip = 0 } = event
  const res = await db.collection(collection)
    .skip(skip)
    .limit(batchSize)
    .get()
  return {
    count: res.data.length,
    data: res.data
  }
}
```

调用：`{ "collection": "leads", "batchSize": 1000, "skip": 0 }`，循环分页获取。

### A.3 导入（批量）

```javascript
// cloudfunctions/import-one-off/index.js
'use strict'
const db = uniCloud.database()
exports.main = async (event) => {
  const { secret, collection, items } = event
  if (secret !== process.env.IMPORT_SECRET) return { code: 401 }
  const BATCH = 500
  let imported = 0
  for (let i = 0; i < items.length; i += BATCH) {
    const batch = items.slice(i, i + BATCH)
    await db.collection(collection).add(batch)
    imported += batch.length
  }
  return { code: 0, imported }
}
```

> ⚠️ `db.collection().add([...])` 单次最多 500 条，超出需自行分批。
